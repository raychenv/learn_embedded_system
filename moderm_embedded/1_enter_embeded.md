# 1 进入嵌入式世界：二进制、补码与第一段 C 代码

## 引言

课程从这一课开始建立最底层的共同语言。后面你会操作寄存器、分析汇编、定位异常，如果没有“数字在机器里如何表示”的基础，很快就会卡住。

第 1 课的核心目标不是写复杂程序，而是建立三个长期有效的能力：

1. 把十进制、十六进制、二进制互相转换，并理解它们只是同一个值的不同写法。
2. 理解二补码（two's complement）如何表示有符号整数。
3. 通过最简单的 C 程序，观察 CPU 在调试器里的真实执行轨迹。

## 为什么嵌入式必须懂进制

在应用层开发中，很多细节被语言和框架屏蔽；在 MCU 开发中，地址、寄存器位、掩码都直接出现在代码里。

例如：

- 地址 `0x400253FC` 是外设寄存器地址。
- 掩码 `0x20` 往往表示某个具体 bit。
- 调试窗口里的数值经常需要在十六进制和二进制之间切换查看。

如果不熟悉进制，看到寄存器内容就像在看乱码。

## 三种常见表示法

同一个值可以有多种表示：

- 十进制：`26`
- 十六进制：`0x1A`
- 二进制：`0b00011010`

对应关系：

$$26 = 1\times16 + 10 = 0x1A$$

而 `0x1A` 再展开为二进制：

$$0x1A = 0001\;1010_2$$

嵌入式里偏爱十六进制，因为它和二进制按 4 位一组天然对应，读寄存器非常高效。

## 位、字节与字长

- 1 bit：二进制一位（0 或 1）
- 1 byte：8 bits
- 32 位 MCU（如 Cortex-M4）：通用寄存器通常 32 位

当你看到 `unsigned int` 在该平台是 32 位时，意味着它可表示：

- 无符号范围：$0$ 到 $2^{32}-1$
- 有符号（二补码）范围：$-2^{31}$ 到 $2^{31}-1$

## 二补码为什么重要

有符号整数在现代 MCU 上几乎都用二补码。它让加减法硬件统一，电路简单且高效。

以 8 位为例：

- `+5`：`00000101`
- `-5`：对 `+5` 逐位取反再加 1：

$$00000101 \xrightarrow{\text{NOT}} 11111010 \xrightarrow{+1} 11111011$$

验证：

$$00000101 + 11111011 = 1\;00000000$$

保留低 8 位后是 `00000000`，结果为 0，符合 `+5 + (-5) = 0`。

这就是第 11 课里讨论有符号/无符号转换风险的数学根基。

## 第一段程序：让 CPU 做可见的工作

lesson-01 源码非常朴素：

```c
int main(void) {
    int volatile counter = 0;
    ++counter;
    ++counter;
    ...
    ++counter;
    return 0;
}
```

看似重复，但教学价值很高：

1. 每一条 `++counter` 都会生成可跟踪的机器指令。
2. 你能在调试器里单步，观察寄存器和变量变化。
3. 程序控制流是完全线性的，便于建立最初的执行模型。

## 为什么这里要用 volatile

如果 `counter` 不是 `volatile`，在高优化等级下编译器可能把一串自增优化掉，因为最终结果可能被认为“无可观察副作用”。

`volatile` 的作用是约束编译器：

- 每次读写都保留。
- 不可随意合并或消除访问。

这在嵌入式里非常常见，后续你会在延时循环、寄存器访问里反复见到它。

## 从 C 到汇编的第一眼

`++counter` 在汇编上通常对应“读-加一-写回”模式，例如概念上：

```asm
LDR   R0, [addr_counter]
ADDS  R0, R0, #1
STR   R0, [addr_counter]
```

你此时不需要记住每条指令细节，重点是形成映射：

- C 语句不是魔法。
- 每一步都有机器级动作。
- 调试器能把这种映射直接呈现出来。

## 调试器里最值得看的三个窗口

第 1 课建议你重点看：

1. Disassembly：当前执行到哪条机器指令。
2. Registers：R0-R15（特别是 PC、SP）。
3. Locals/Watch：变量值如何随着单步变化。

只要这三个窗口能看懂，后面讲栈、函数调用、异常处理时你就不会“只看源码猜行为”。

## PC、顺序执行与“下一条指令”

程序计数器 PC 指向下一条待执行指令地址。线性代码中，PC 大多按固定步长向前推进；遇到分支、函数调用、异常时才会跳转。

第 1 课的线性程序正好用来建立这个直觉：

- 大部分时间 PC 递增。
- 没有分支时控制流最简单。

这将直接过渡到第 2 课的控制流分支（`if/else`、`while`）。

## 常见误区

### 1. 以为十六进制是“另一种数”

不是。它只是同一个值的表示法，和十进制地位等价。

### 2. 以为负数是“额外加个符号位”

在硬件实现上，二补码不是简单符号位拼接，而是一整套编码规则。

### 3. 以为编译器总会“照着源码逐行做”

只有在特定设置（低优化/volatile）下更接近逐行。优化器会重排、折叠甚至消除代码。

## 这一课在全课程中的位置

你现在建立的是“机器可解释性”基础：

- 数值表示：看得懂 bit 模式。
- 变量更新：知道会变成哪些底层动作。
- 调试方法：会用窗口观察而不是猜。

第 2 课会在这个基础上加入分支和循环控制流；第 3 课再引入变量作用域、指针和内存地址。到第 4 课，你就会真正开始访问外设寄存器点灯。

## 小结

第 1 课完成了课程最底层的认知搭建：

1. 进制转换和十六进制阅读能力。
2. 二补码下有符号整数的表示逻辑。
3. 第一段 C 程序在调试器中的执行观察方法。

从这里出发，你后续学到的每一个寄存器地址、每一个位掩码、每一个异常栈帧，都能落回这套基础解释框架。
