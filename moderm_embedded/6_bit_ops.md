# 6 位运算驱动 GPIO：只改你想改的那一位

## 引言

第 5 课我们已经把代码写得更可维护，但仍有一个核心问题没有解决：

- 如果直接给 GPIO 数据寄存器赋值，往往会覆盖其他位。
- 你想“点红灯不动蓝灯”，却可能把蓝灯状态清掉。

第 6 课要解决的就是这个工程痛点：**用位运算做按位控制**。

本课同时建立两个能力：

1. 理解 C 位运算符在机器层的真实行为。
2. 把这些运算应用到多路 LED 控制，避免破坏无关位。

## Lesson-06 代码锚点

TM4C 版本核心片段：

```c
#define LED_RED   (1U << 1)
#define LED_BLUE  (1U << 2)
#define LED_GREEN (1U << 3)

SYSCTL_RCGCGPIO_R |= (1U << 5);
GPIO_PORTF_DIR_R  |= (LED_RED | LED_BLUE | LED_GREEN);
GPIO_PORTF_DEN_R  |= (LED_RED | LED_BLUE | LED_GREEN);

GPIO_PORTF_DATA_R &= ~(LED_RED | LED_BLUE | LED_GREEN);
GPIO_PORTF_DATA_R |= LED_BLUE;

while (1) {
    GPIO_PORTF_DATA_R |= LED_RED;
    ...delay...
    GPIO_PORTF_DATA_R &= ~LED_RED;
    ...delay...
}
```

你会看到两类关键写法：

- 置位：`reg |= mask`
- 清位：`reg &= ~mask`

这是嵌入式里最基础也最常用的按位更新模板。

## 位运算符速览

本课涉及的运算符：

1. `|`：按位或（OR）
2. `&`：按位与（AND）
3. `^`：按位异或（XOR）
4. `~`：按位取反（NOT）
5. `<<`：左移
6. `>>`：右移

课程源码里还做了实验变量：

```c
unsigned int a = 0x5A5A5A5A;
unsigned int b = 0xDEADBEEF;
unsigned int c;

c = a | b;
c = a & b;
c = a ^ b;
c = ~b;
c = a << 1;
c = b >> 3;
```

这部分的价值在于：你可以在调试器里看二进制位如何变化，再对应汇编指令。

## 为什么位运算是 GPIO 控制的核心

假设寄存器当前状态：

- 位 2（蓝灯）是 1
- 位 1（红灯）是 0

你想打开红灯，若写：

```c
GPIO_PORTF_DATA_R = LED_RED;
```

结果是整个寄存器被覆盖，蓝灯位可能丢失。

正确写法：

```c
GPIO_PORTF_DATA_R |= LED_RED;
```

这条语句等价于：

$$reg \leftarrow reg \;|\; mask$$

- `mask` 中为 1 的位被置 1。
- 其他位保持原值。

同理，关闭红灯用：

```c
GPIO_PORTF_DATA_R &= ~LED_RED;
```

等价于：

$$reg \leftarrow reg \;&\; \sim mask$$

- `mask` 中为 1 的位被清 0。
- 其他位保持原值。

## 三个高频模板（必须熟练）

设 `reg` 是寄存器，`mask` 是目标位掩码：

1. 置位：`reg |= mask`
2. 清位：`reg &= ~mask`
3. 翻转：`reg ^= mask`

这是绝大多数寄存器配置代码的核心语法。

## 左移和位掩码命名

位定义常用左移表达：

```c
#define LED_RED   (1U << 1)
#define LED_BLUE  (1U << 2)
#define LED_GREEN (1U << 3)
```

优点：

1. 一眼看出位号（bit position）。
2. 减少手写 hex 的错误概率。
3. 易于组合：`LED_RED | LED_GREEN`。

在硬件寄存器编程里，这是比直接写 `0x02U/0x04U/0x08U` 更好的风格。

## 右移在有符号数上的关键差异

lesson-06 还演示了：

```c
int x = 1024;
int y = -1024;
int z;

z = x >> 3;
z = y >> 3;
```

对无符号数，右移通常是逻辑右移（高位补 0）；
对有符号数，很多编译器/架构会执行算术右移（高位补符号位），以保持符号。

在二补码下，这能让“右移约等于除以 $2^n$”在负数上也维持直觉上的行为。但严格标准语义与实现相关，跨平台代码应谨慎依赖。

## 位运算到汇编：一条指令改 32 位

在 Cortex-M 上，常见映射如下：

- `|` → `ORR`
- `&` → `AND`
- `^` → `EOR`
- `~` → `MVN`
- `<<` / `>>` → `LSL` / `LSR`（或算术右移 `ASR`）

这意味着你写一行 C 位运算，CPU 往往只执行一条 ALU 指令就完成 32 位并行处理，效率极高。

## STM32 版本的对应思路

lesson-06 的 STM32C031 版本同样体现“位掩码 + 按位配置”逻辑，例如：

```c
#define LD4_PIN  5U
#define LD5_PIN  6U

GPIOA_MODER_R &= ~((3U << 2U*LD4_PIN) | (3U << 2U*LD5_PIN));
GPIOA_MODER_R |=  ((1U << 2U*LD4_PIN) | (1U << 2U*LD5_PIN));
...
GPIOA_BSRR_R = (1U << LD5_PIN);
```

这里 `3U << 2*pin` 这种写法，本质是在“每个引脚两位模式字段”中精准定位并修改对应位段。

## 注意：读改写与并发上下文

`reg |= mask` 在 C 层面是“读-改-写”三步。如果寄存器可能被中断或其他上下文同时修改，可能出现竞态。

解决手段依平台而定：

1. 使用硬件提供的原子置位/清位寄存器（如 STM32 的 BSRR）。
2. 在关键区短暂关中断。
3. 使用位带（bit-banding，若芯片支持）。

本课主要建立位运算语义，后续课程会逐步扩展到更严格的实时并发安全。

## 为什么先做“二进制实验”

很多同学觉得 `a|b`、`a&b` 是“课堂知识”，但在嵌入式它是日常基本功。

课程先让你在普通变量上看位变化，再迁移到 GPIO 寄存器，目的就是让你形成稳定映射：

- 位运算不是抽象数学。
- 每一位都可能连接真实引脚和真实电平。
- 运算结果立即体现在硬件行为上。
